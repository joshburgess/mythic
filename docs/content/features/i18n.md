# Internationalization (i18n)

Mythic provides first-class support for multilingual sites. You can organize content by locale, serve translated pages, generate hreflang tags for SEO, and access translations in templates.

## Configuration

Enable i18n in `mythic.toml`:

```toml
[i18n]
default_locale = "en"
locales = ["en", "fr", "de", "ja"]
```

| Option           | Type   | Default  | Description                               |
|------------------|--------|----------|-------------------------------------------|
| `default_locale` | String | `"en"`   | The primary locale                        |
| `locales`        | Array  | `["en"]` | All supported locales                     |

## Content Organization

### Locale Directories

The recommended approach is to organize content into locale subdirectories:

```
content/
  en/
    index.md
    about.md
    blog/
      first-post.md
  fr/
    index.md
    about.md
    blog/
      first-post.md
  de/
    index.md
    about.md
```

Pages in locale directories automatically have their locale set. The URL structure reflects the locale:

| File                            | URL              |
|---------------------------------|------------------|
| `content/en/about.md`          | `/en/about/`     |
| `content/fr/about.md`          | `/fr/about/`     |
| `content/de/about.md`          | `/de/about/`     |

The default locale can optionally omit the locale prefix:

```toml
[i18n]
default_locale = "en"
hide_default_locale = true
```

With this setting, English pages are served without the `/en/` prefix:

| File                            | URL              |
|---------------------------------|------------------|
| `content/en/about.md`          | `/about/`        |
| `content/fr/about.md`          | `/fr/about/`     |

### Frontmatter Locale

Alternatively, set the locale directly in frontmatter:

```yaml
---
title: "A propos"
locale: fr
---
```

This works when you prefer a flat content directory rather than locale subdirectories.

## Translation Files

Create translation files in `_data/translations/`:

```
_data/
  translations/
    en.yaml
    fr.yaml
    de.yaml
    ja.yaml
```

Each file contains key-value pairs:

```yaml
# _data/translations/en.yaml
nav:
  home: "Home"
  about: "About"
  blog: "Blog"
  contact: "Contact"
footer:
  copyright: "All rights reserved."
  built_with: "Built with Mythic"
read_more: "Read more"
posted_on: "Posted on"
tagged_with: "Tagged with"
```

```yaml
# _data/translations/fr.yaml
nav:
  home: "Accueil"
  about: "A propos"
  blog: "Blog"
  contact: "Contact"
footer:
  copyright: "Tous droits reserves."
  built_with: "Construit avec Mythic"
read_more: "Lire la suite"
posted_on: "Publie le"
tagged_with: "Etiquete avec"
```

## The t() Function

Use the `t()` function in templates to access translations for the current page's locale:

```html
<!-- Tera -->
<nav>
    <a href="/">{{ t(key="nav.home") }}</a>
    <a href="/about/">{{ t(key="nav.about") }}</a>
    <a href="/blog/">{{ t(key="nav.blog") }}</a>
</nav>

<footer>
    <p>{{ t(key="footer.copyright") }}</p>
</footer>
```

```html
<!-- Handlebars -->
<nav>
    <a href="/">{{t "nav.home"}}</a>
    <a href="/about/">{{t "nav.about"}}</a>
</nav>
```

### Fallback Behavior

If a translation key is missing for the current locale, Mythic falls back to the default locale. If the key is missing there too, the key itself is displayed.

### Interpolation

Translations support variable interpolation:

```yaml
# _data/translations/en.yaml
greeting: "Hello, {name}!"
post_count: "{count} posts"
```

```html
{{ t(key="greeting", name=data.site.author) }}
{{ t(key="post_count", count=site.sections.blog | length) }}
```

## Hreflang Tags

Mythic automatically generates hreflang `<link>` tags for pages that have translations. Add them in your base template:

```html
<head>
    <!-- Other head elements -->

    {% if page.translations %}
    <link rel="alternate" hreflang="{{ page.locale }}" href="{{ site.base_url }}{{ page.path }}">
    {% for translation in page.translations %}
    <link rel="alternate" hreflang="{{ translation.locale }}" href="{{ site.base_url }}{{ translation.path }}">
    {% endfor %}
    <link rel="alternate" hreflang="x-default" href="{{ site.base_url }}{{ page.path }}">
    {% endif %}
</head>
```

Mythic links pages as translations when they share the same relative path across locale directories.

## Language Switcher

Build a language switcher in your templates:

```html
<div class="language-switcher">
    {% if page.translations %}
    <span>{{ page.locale | upper }}</span>
    {% for translation in page.translations %}
        <a href="{{ translation.path }}">{{ translation.locale | upper }}</a>
    {% endfor %}
    {% endif %}
</div>
```

A more detailed switcher with language names:

```html
{% set locale_names = ["en:English", "fr:Francais", "de:Deutsch", "ja:Japanese"] %}

<nav class="lang-nav" aria-label="Language">
    {% for translation in page.translations %}
    <a href="{{ translation.path }}" lang="{{ translation.locale }}">
        {{ translation.locale_name }}
    </a>
    {% endfor %}
</nav>
```

## Locale-Specific Data

Data files can be organized by locale:

```
_data/
  en/
    nav.yaml
  fr/
    nav.yaml
```

Access locale-specific data:

```html
{% for item in data[page.locale].nav.items %}
    <a href="{{ item.url }}">{{ item.label }}</a>
{% endfor %}
```

## RTL Support

For right-to-left languages, set the direction in your base template:

```html
<html lang="{{ page.locale }}" dir="{{ t(key='direction') | default(value='ltr') }}">
```

With translation files:

```yaml
# _data/translations/ar.yaml
direction: "rtl"
```

```yaml
# _data/translations/en.yaml
direction: "ltr"
```

## Feed Generation per Locale

Feeds are generated per locale when i18n is enabled:

```
public/
  en/
    atom.xml          # English feed
  fr/
    atom.xml          # French feed
```

Each feed contains only pages in that locale.
